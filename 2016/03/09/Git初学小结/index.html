Git初学小结

发表于 2016-03-09

Git学习

入门教程:史上最浅显易懂的Git教程

基本命令

  • git init
    创建版本库
  • git add
    提交变更到暂存区
  • git commit
    提交变更到版本库
  • git status
    查看仓库当前的状态
  • git diff
    查看修改的内容
  • git log
    显示从最近到最远的提交日志，若嫌输出信息太多，可以加上--pretty=oneline参数
  • git reset
    版本回退，加上 commit id，版本号没必要写全，git 会自动去找。版本回退采用指向当前版本的HEAD指针，HEAD 指向的版本就是当前版本，git 允许我们在版本的历史之间穿梭，使用命令 git reset --hard commit_id
  • git reflog
    查看命令历史 ，可以查看 commit_id
  • git checkout -- file
    丢弃工作区的修改，没有–命令就变成了“切换到另一个分之”
  • git rm
    删除版本库中的文件

远程仓库相关操作

  1. 创建SSH Key
    ssh-keygen -t rsa -C "YourEmail@example.com" 一路回车使用默认值即可。一切顺利，可以在用户主目录里找到.ssh目录，里面有id_rsa和id_rsa.pub两个文件，SSH Key的秘钥对，id_rsa是私钥，不能泄露，id_rsa.pub是公钥。
  2. 登录GitHub
    打开”Account Settings”->”SSH Keys”页面，添加SSH Key
  3. 添加远程库
    在GitHub上新建仓库，根据GitHub的提示，在本地仓库目录下运行命令：git remote add origin git@github.com:UserName/RepositoryName.git
  4. git push -u origin master
    远程库的名字就是origin，Git默认叫法，此命令把本地卡的所有内容推送到远程库上，第一次推送master分支的所有内容
  5. git push origin master
    每次本地提交后，使用此命令推送
  6. 克隆远程库
    git clone git@github.com:UserName/RepositoryName.git

分支相关操作

  • git checkout -b xxx
    创建xxx分支库并且切换到xxx分支，加上-b参数相当于以下两条命令
    git branch xxx
    git checkout xxx
  • git branch
    列出所有分支，当前分支前面会标一个*号
  • git checkout xxx
    切换到xxx分支
  • git merge xxx
    合并xxx分支
  • git branch -d xxx
    删除xxx分支
  • Git会用<<<<<<<,=======,>>>>>>>标记出冲突文件中不同分支的内容
  • git log --graph
    查看分支合并图
  • git merge
    合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，而fast forward合并则看不出来曾经做过合并
  • git stash
    “储藏”当前工作现场，等待恢复
  • git stash list
    查看储藏的列表
  • git stash apply
    进行工作现场恢复，但是恢复后stash内容并不删除，需要用 git stash drop 来删除
  • git stash pop
    恢复工作现场的同时把stash的内容也删了
  • git branch -D xxx
    强行删除xxx分支
  • git remote -v
    查看远程库信息
  • 本地新建的分支如果不推送到远程，对其他人就是不可见的
  • 从本地推送分支，使用git push origin branch-name，如果推送失败，先用 git pull 抓取远程的新提交
  • 在本地创建和远程分支对应的分支，使用 git checkout -b branch-name origin/branch-name ，本地和远程分支的名称最好一致
  • 建立本地分支和远程分支的关联，使用 git branch --set-upstream branch-name origin/branch-name
  • 从远程抓取分支，使用 git pull ，如果有冲突，要先处理冲突

标签相关

  • git tag <name> 用于新建一个标签，默认为 HEAD ，也可以指定一个commit id
  • git tag -a <name> -m "blablabla..." 指定标签信息
  • git tag -s <name> -m "blablabla..." 用PGP签名标签
  • git tag 查看所有标签
  • git push origin <name> 推送一个本地标签
  • git push origin --tags 推送全部未推送过的本地标签
  • git tag -d <name> 删除一个本地标签
  • git push origin :refs/tags/<name> 删除一个远程标签
